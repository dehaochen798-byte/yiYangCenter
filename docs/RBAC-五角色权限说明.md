# 五角色 RBAC 说明

## 标准角色

- 管理员
- 护理主管
- 护理人员
- 前台人员
- 膳食管理员

## 历史岗位兼容映射

- `系统管理员 -> 管理员`
- `护士长 -> 护理主管`
- `护理员 -> 护理人员`
- `健康管家 -> 前台人员`

## 模块权限矩阵

| 角色 | 工作台 | 员工账号 | 客户档案 | 护理级别/内容 | 护理记录 | 床位/入住/外出/退住 | 膳食计划/日历 | 服务对象分配 | 服务关注 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 管理员 | 可见 | CRUD | CRUD | CRUD | CRUD | CRUD | CRUD | CRUD | CRUD |
| 护理主管 | 可见 | 无 | 只读全部 | CRUD | CRUD | 无 | 无 | CRUD | 无 |
| 护理人员 | 可见 | 无 | 仅负责老人只读 | 只读 | 仅本人记录增改查，不可删 | 无 | 无 | 无 | 仅负责老人只读 |
| 前台人员 | 可见 | 无 | 只读全部 | 无 | 无 | CRUD | 无 | 无 | 无 |
| 膳食管理员 | 可见 | 无 | 只读全部 | 无 | 无 | 无 | CRUD | 无 | 无 |

## 负责老人规则

- 护理人员的“负责老人”来源于 `ServiceTarget.managerUserId`
- 若 `startDate` 不为空，则要求 `startDate <= 当前日期`
- 若 `endDate` 不为空，则要求 `endDate >= 当前日期`
- 起止日期为空时视为长期有效

## 前端行为

- 侧边菜单、路由守卫、工作台快捷入口按角色裁剪
- 无权直达页面统一进入 `403`
- `客户档案` 与 `员工账号` 已拆成两个入口
- 护理人员进入护理记录页时，执行人默认锁定为本人

## 后端行为

- gateway 先做模块级准入拦截
- `service-care` 再做细粒度角色与数据范围校验
- 仅前端隐藏不会放行接口，请以接口权限为最终准则
